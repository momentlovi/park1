# МФК ПАРК  Статичный сайт для GitHub Pages

Лендинг премиального многофункционального комплекса в Сочи.
Стек: HTML5, CSS3 (Variables, Flexbox/Grid), Vanilla JS (ES6+).
Без использования сторонних библиотек и сборщиков.

##  Структура проекта

``r
/
 index.html          # Главная страница (SEO, Schema.org, Content)
 404.html            # Страница ошибки (для корректной работы GitHub Pages)
 robots.txt          # Инструкции для поисковых роботов
 sitemap.xml         # Карта сайта
 README.md           # Документация
 assets/
     css/
        styles.css  # Все стили (адаптив, анимации, шрифты)
     js/
        main.js     # Скрипты (меню, скролл, лайтбокс, аккордеон)
     img/            # Изображения (нужно заменить плейсхолдеры)
     fonts/          # Локальные шрифты (опционально, сейчас Google Fonts)
``r

##  Инструкция по деплою на GitHub Pages

1. **Создайте репозиторий** на GitHub (например, park-sochi).
2. **Загрузите файлы** проекта в этот репозиторий (можно через git push или веб-интерфейс Upload files).
3. Перейдите в **Settings** (Настройки) репозитория.
4. В меню слева выберите раздел **Pages**.
5. В блоке **Build and deployment**:
   - Source: Deploy from a branch`r
   - Branch: main (или master) / folder: /(root)`r
6. Нажмите **Save**.
7. Через 1-2 минуты сайт будет доступен по адресу: https://ваш-username.github.io/park-sochi/.

##  Настройка контента

### 1. Ссылки и контакты
Откройте index.html и найдите:
- 	el:+70000000000  замените на реальный телефон.
- mailto:info@park-sochi.ru  замените на реальную почту.
- Ссылки на соцсети (WhatsApp, Telegram).

### 2. SEO
В index.html, sitemap.xml и obots.txt замените your-username.github.io/park-sochi на реальный URL вашего сайта.

### 3. Изображения
Замените изображения в папке ssets/img/ или обновите ссылки в index.html. Сейчас используются плейсхолдеры placehold.co.
Рекомендуемые форматы: WebP или оптимизированный JPG.

##  Оптимизация (Performance)
- Используется loading=lazy` для изображений.
- Шрифты загружаются через preconnect.
- CSS и JS минифицированы (по структуре готовы к минификации).
- Отсутствуют тяжелые библиотеки.
